# D6 — Skills via harvest: staged, project-scoped, high bar

Date: 2026-07-04 · Owner-approved

## Decision

- Mistakes → skills happens in a HARVEST phase (Phase 3, post-feature),
  by a librarian session — never mid-loop (one-story scope, and one
  fresh incident = worst moment to generalize)
- Skills STAGED in `.claude/skills-proposed/`; human approves to live.
  Never auto-live
- Project scope only (`.claude/skills/`); global skills = personal,
  human-curated, not automated
- Promotion bar: (1) cost ≥2 iterations or recurred ≥2 stories, AND
  (2) procedural countermeasure, AND (3) one-line trigger. Else
  learnings page

## Rationale

- Wrong skill worse than no skill: auto-fires in every future session,
  silently steers all work
- Every live skill = permanent context tax on all sessions → high bar;
  50 skills = 50-page-index disease
- Owner goal: minimize always-on context; load knowledge only when
  needed. Push (skill) reserved for procedures that MUST not be
  forgotten; pull (learnings) for everything else

## Owner note (startup mode)

Make it work first, optimize cost later. v1 quality bar = correctness
of promotion, not token metering.

## Revisit when

- Approved skills rarely fire → descriptions too narrow, or bar too
  high in practice
- Skill count grows past ~10 in one project → audit for merges, raise bar
- Same mistake still recurs WITH a live skill → skill body weak, or
  trigger description not matching real situations
